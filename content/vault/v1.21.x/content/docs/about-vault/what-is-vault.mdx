# O que é o Vault?

O Vault oferece acesso privilegiado centralizado e auditado, além de
gerenciamento de segredos para dados críticos, independentemente de você
implantar sistemas localmente, na nuvem ou em um ambiente híbrido.

Com um design modular baseado em um ecossistema de plugins em crescimento, o
Vault permite que você o integre aos seus sistemas existentes e personalize o
fluxo de trabalho da sua aplicação.

## Por que devo usar o Vault?

Softwares modernos funcionam por causa de **segredos**.
Segredos são informações confidenciais e discretas, como credenciais, chaves de
criptografia, certificados de autenticação e outras informações críticas que
suas aplicações precisam para funcionar de forma consistente e segura.

O Vault ajuda a fortalecer a segurança das aplicações centralizando o
gerenciamento de segredos.
Com o Vault, você pode:

- [Gerenciar segredos estáticos](/vault/docs/about-vault/why-use-vault/static-secrets)
- [Gerenciar certificados](/vault/docs/about-vault/why-use-vault/certificates)
- [Gerenciar identidades e autenticação](/vault/docs/about-vault/why-use-vault/identities)
- [Gerenciar segredos de terceiros](/vault/docs/about-vault/why-use-vault/3rd-party-secrets)
- [Gerenciar dados sensíveis](/vault/docs/about-vault/why-use-vault/sensitive-data)
- [Suportar conformidade regulatória](/vault/docs/about-vault/why-use-vault/regulatory-compliance)

## O que é um plugin?

Os plugins funcionam como blocos de construção no Vault que você controle como
os dados se movem pelo seu ambiente e como os clientes acessam esses dados.

O [ecossistema de plugins](/vault/docs/plugins) inclui:

- Plugins de autenticação que gerenciam fluxos de autenticação e controlam o
  acesso do cliente ao Vault;
- Plugins de segredos em geral que geram, armazenam, gerenciam ou transformam
  informações confidenciais;
- Plugins de segredos de banco de dados que gerenciam as credenciais dinâmicas
  que os clientes usam para acessar dados do banco de dados.

Use plugins do [registro de plugins selecionados](/vault/integrations) ou
[crie plugins personalizados](/vault/docs/plugins/plugin-development) para
integrar o Vault da maneira que fizer mais sentido para seus fluxos de trabalho.

## Quem pode acessar os dados no Vault?

O Vault criptografa os dados em repouso e controla o acesso a esses dados com
métodos configuráveis e robustos de [autenticação](/vault/docs/concepts/auth) e
[autorização](/vault/docs/concepts/policies).

![Como o Vault funciona](/img/how-vault-works.png)

1. Os clientes se autenticam com tokens gerados manualmente, protocolos como
   LDAP ou provedores terceirizados como Azure e AWS;

2. O Vault gera um token de acesso que vincula a requisição do cliente a uma
   entidade interna e às políticas de segurança aplicáveis;

3. Os clientes interagem com segredos e operações de criptografia com base em
   caminhos de recursos montados no Vault;

4. O Vault autoriza a requisição do cliente de acordo com as políticas definidas
   no caminho de recurso e concede ou nega o acesso conforme necessário.

Durante todo o processo, o Vault audita todas as atividades, independentemente
de a autenticação ou autorização ser bem-sucedida, para que você possa rastrear
as interações com os sistemas de missão crítica.

## Onde o Vault armazena os dados?

O Vault suporta diversas opções de armazenamento durável de informações.

Tipo de armazenamento                                               | Suporte a alta disponibilidade | Descrição
------------------------------------------------------------------- |:------------------------------:|----------------------------------------------------------------------------------------------------------------
[Integrado](/vault/docs/configuration/storage/raft)                 |              SIM               | A opção de armazenamento "integrada" que criptografa e replica dados em um cluster do Vault em operação.
[Sistema de arquivos](/vault/docs/configuration/storage/filesystem) |              NÃO               | Persiste os dados no sistema de arquivos local da máquina que executa o Vault.
[Externo](/vault/docs/configuration/storage#integrated-vs-external) |             TALVEZ             | Um sistema de armazenamento durável de terceiros, como Azure, AWS, Google Cloud ou MySQL.
[Em memória](/vault/docs/configuration/storage/in-memory)           |              NÃO               | Persiste os dados inteiramente na memória da máquina que executa o Vault para desenvolvimento e experimentação.

Recomendamos o armazenamento integrado para a maioria das implantações.
O armazenamento integrado suporta fluxos de trabalho de backup/restauração, alta
disponibilidade e recursos de replicação corporativa sem depender de sistemas de
terceiros nos quais o Vault não pode verificar a segurança e a rastreabilidade
do acesso aos dados.

## Quando não devo usar o Vault?

O Vault é robusto, poderoso e flexível.
Mas também pode ser complexo demais se você tiver necessidades limitadas ou
simples de gerenciamento de segredos.

Se sua organização está apenas começando com o gerenciamento de segredos ou
procurando simplificar os processos de gerenciamento de segredos existentes,
considere começar com o [HCP Vault Dedicated](/hcp/docs/vault/what-is-hcp-vault)
em vez do Vault.

O HCP Vault Dedicated é uma oferta gerenciada do Vault que roda na plataforma de
nuvem da HashiCorp.
O HCP Vault Dedicated fornece um cluster Vault Enterprise sem a sobrecarga
operacional de planejar, implantar e gerenciar um cluster Vault com hospedagem
própria.

## Como obtenho o Vault?

Você pode
[baixar o Vault como um binário pré-compilado](http://releases.hashicorp.com/vault),
instalar um pacote oficial [Community](/vault/install) ou
[Enterprise](/vault/install/enterprise) com gerenciadores de pacotes
compatíveis, ou clonar o repositório do Vault Community no GitHub e
[compilar o Vault a partir do código-fonte](/vault/get-vault/build-from-code).

Para usar os recursos do Vault Enterprise, você precisa ter uma
[licença válida configurada](/vault/license).
